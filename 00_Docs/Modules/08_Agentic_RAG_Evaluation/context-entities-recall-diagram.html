Context Entities Recall

Measures what fraction of the named entities in the reference are also present in the retrieved context.

Input 1
Ground-Truth Reference
Input 2
Retrieved Context
⚡

No LLM needed! Unlike Context Recall, this metric uses pure entity extraction (NER) — no LLM judgement calls. It compares entity sets directly, making it faster and fully deterministic.

1
Extract
Find all entities in the reference (RE)
Extract every named entity — people, places, dates, organizations, etc. — from the ground-truth reference.
Reference "Albert Einstein developed the theory of relativity at the ETH Zurich. He was born in Ulm, Germany in 1879."

→ Entities in Reference (RE) Albert Einstein ETH Zurich Ulm Germany 1879
2
Extract
Find all entities in the retrieved context (RCE)
Do the same entity extraction on the retrieved context.
Retrieved Context "Einstein is famous for the theory of relativity. He was born in 1879 and later worked at Princeton."

→ Entities in Context (RCE) Einstein 1879 Princeton
3
Intersect
Find common entities between RE and RCE
Compare the two entity sets. Which entities from the reference also appear in the retrieved context?
→ Match check against RE
Albert Einstein ✓ ETH Zurich ✗ Ulm ✗ Germany ✗ 1879 ✓
Common (RE ∩ RCE): 2  |  Total in RE: 5
4
Compute
Calculate Context Entity Recall
Divide the count of common entities by the total entities in the reference. Result is 0 (no entities recovered) to 1 (all entities recovered).
CER =
| RE ∩ RCE | | RE |
RE = entities in reference RCE = entities in retrieved context
Worked Example
2 5
= 0.40
