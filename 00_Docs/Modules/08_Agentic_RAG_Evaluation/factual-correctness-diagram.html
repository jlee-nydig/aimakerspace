Factual Correctness

Measures the factual overlap between the generated response and the ground-truth reference using precision, recall, and F1 score. Higher is better (0 → 1).

Input 1
Generated Response
Input 2
Ground-Truth Reference
⚖️

Two-way comparison! Unlike Faithfulness (which only checks the answer against context), Factual Correctness decomposes both the response and the reference into claims, then cross-checks in both directions — yielding TP, FP, and FN for a full precision / recall / F1 picture.

1
Decompose
Break both response and reference into claims
An LLM reads the generated response and the ground-truth reference separately, splitting each into atomic factual claims.
Response "The Eiffel Tower is in Paris. It was built in 1889. It is 500m tall."

→ Response claims
① The Eiffel Tower is in Paris
② It was built in 1889
③ It is 500m tall

Reference "The Eiffel Tower is in Paris. It was built in 1889. It is 330m tall. It was designed by Gustave Eiffel."

→ Reference claims
① The Eiffel Tower is in Paris
② It was built in 1889
③ It is 330m tall
④ It was designed by Gustave Eiffel
2
Cross-check
Determine factual overlap using NLI
Using natural language inference, check each response claim against the reference (and vice versa) to find what's shared, what's extra, and what's missing.
Response → Reference
✓
Eiffel Tower is in Paris — in reference
✓
Built in 1889 — in reference
✗
500m tall — contradicts reference (330m)

Reference → Response
✓
Eiffel Tower is in Paris — in response
✓
Built in 1889 — in response
✗
330m tall — response says 500m
✗
Designed by Gustave Eiffel — missing from response
3
Classify
Assign TP, FP, and FN
Based on the cross-check, classify all claims into three buckets.
True Positive (TP)
2
Response claims present in reference
False Positive (FP)
1
Response claims NOT in reference
False Negative (FN)
2
Reference claims NOT in response
4
Compute
Calculate Precision, Recall, and F1
Use TP, FP, FN to compute Precision (how much of the response is correct), Recall (how much of the reference is covered), and the F1 Score (harmonic mean of both).
Precision =
TP TP + FP
Recall =
TP TP + FN
F1 Score =
2 × Precision × Recall Precision + Recall
TP = response claims present in reference
FP = response claims NOT in reference
FN = reference claims NOT in response
Available Modes
Precision
"Is what the response says correct?"
Penalizes hallucinations
Recall
"Does the response cover everything?"
Penalizes omissions
Default
F1 Score
Balanced harmonic mean
of precision & recall
Worked Example
TP = 2   FP = 1   FN = 2
Precision = 2 / (2+1) = 0.67
Recall = 2 / (2+2) = 0.50
F1 = 2 × 0.67 × 0.50 / (0.67 + 0.50)
0.67 1.17
= 0.57
